```
0:00-0:03  Visual hook — result or bold claim
0:03-0:35  One concept, one explanation, one demonstration
0:35-0:45  Takeaway + "Full tutorial on channel"
```

## Blog-to-Video Pipeline

### Operational Flow

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  Research        │────▶│  Blog Post       │────▶│  Video Script   │
│  (Grok/Gemini)  │     │  (blog pipeline) │     │  (adapt from    │
│                  │     │                  │     │  blog content)  │
└─────────────────┘     └──────────────────┘     └─────────────────┘
                                                          │
                              ┌────────────────────────────┘
                              ▼
                    ┌─────────────────┐     ┌─────────────────┐
                    │  Record Video   │────▶│  Edit + Publish │
                    │  (OBS + Zeus)   │     │  (DaVinci/      │
                    │                 │     │   CapCut)       │
                    └─────────────────┘     └─────────────────┘
                                                    │
                              ┌──────────────────────┘
                              ▼
                    ┌─────────────────┐     ┌─────────────────┐
                    │  Cut Shorts     │     │  Update Blog    │
                    │  (2-3 per video)│     │  (embed video,  │
                    │                 │     │   add link)     │
                    └─────────────────┘     └─────────────────┘
```

### Script Adaptation Process

1. **Blog post exists** (already written, published, SEO'd)
2. **Extract the hook** — first 10% becomes YouTube opening (bold claim, result, question)
3. **Plan whiteboard segments** — which concepts need physical drawing to demonstrate teaching?
4. **Plan screen recording** — which parts are live coding / terminal / browser demo?
5. **Write transitions** — conversational bridges between segments
6. **Plan the "try this yourself"** — homework/challenge at the end
7. **Record** — OBS on Zeus (Pop!_OS), webcam + screen capture + whiteboard camera
8. **Edit** — DaVinci Resolve (free, available on Linux)
9. **Publish** — YouTube with chapters, description linking to blog, SEO keywords
10. **Cut Shorts** — 2-3 best 30-60s moments from the video
11. **Cross-link** — Update blog post with embedded YouTube video

### Production Setup (Zeus Workstation)

| Component | Tool | Notes |
|-----------|------|-------|
| Screen recording | OBS Studio | Multi-scene: code + face PiP + whiteboard |
| Video editing | DaVinci Resolve | Free, Linux native, professional grade |
| Webcam | Logitech C920 or similar | 1080p, good low-light |
| Microphone | Blue Yeti / AT2020 | Clear audio is non-negotiable |
| Lighting | Ring light or softbox | Clean, professional look |
| Whiteboard | Physical small whiteboard | Mounted behind/beside desk, camera angle on it |
| Thumbnails | Canva or GIMP | Consistent template: face + bold text + visual |
| Shorts editing | CapCut (free) | Quick cuts, text overlays, vertical format |

## Key Decisions & Principles

1. **One video per week minimum** — consistency over perfection
2. **Blog post always comes first** — it's the "textbook chapter" the video illustrates
3. **Every video must work for both audiences** — a student can learn, a dev can enjoy
4. **Show your work, including failures** — Rossmann principle: process > polish
5. **Whiteboard at least once per video** — proves physical classroom capability
6. **Clean language always** — the entire channel must be parent-safe and school-safe
7. **No hustle energy ever** — never sell, never hype, never promise riches
8. **Projects must produce portfolio artifacts** — students leave each video with something built
9. **Fun is mandatory** — if it's not interesting to you, don't make the video
10. **This is a marathon, not a sprint** — 24-month timeline, compounding consistency
